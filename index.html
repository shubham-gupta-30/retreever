Motivation

Dense retrieval is fast but opaque. Hierarchical retrieval is interpretable but slow.

ReTreever learns a binary tree end-to-end for retrieval. A frozen encoder produces query and document embeddings, then differentiable split nodes route both inputs through a shared tree. The resulting node-assignment vectors act as coarse-to-fine representations, so the same trained model can trade representation size, latency, and accuracy at inference time.

01

Runs on embeddings

No LLM calls are required to build or navigate the hierarchy, making tree-based retrieval practical beyond small corpora.

02

Coarse-to-fine retrieval

Use any tree level as an embedding: coarse levels are cheaper, leaves are finer, and one model supports both.

03

Inspectable structure

Tree paths and intermediate nodes reveal semantic groupings, providing a window into why content is retrieved.

Method

A tree learned for retrieval

A split function at each node outputs a soft left/right routing probability. Probabilities multiply along the path to produce a distribution over leaves. A bidirectional contrastive objective pulls positive query-reference pairs toward the same leaf assignment while separating negatives.

1

Encode

Independently embed queries and reference contexts with a frozen encoder.

2

Route

Pass embeddings through a differentiable binary tree organizer.

3

Match

Compare leaf-assignment vectors using the training-aligned distance.

4

Inspect

Use tree paths and intermediate nodes as semantic groupings.

Scale

Scales to 124.5M documents

On the Reddit Title-Body corpus, the learned hierarchy provides a memory-friendly multi-index alternative to global approximate nearest-neighbor search.

Interpretability

Semantic trees you can inspect

The learned tree can be probed across modalities: text subtrees reveal topical refinement, while image subtrees reveal visually coherent groups.

A

Transparent organization

Nodes group content semantically, even though training optimizes retrieval rather than clustering.

B

Inspectable retrieval

Routing paths give a compact view of how a query reaches a semantic region.

C

Flexible search

Use single-index search at a chosen level or multi-index search over top-k routed leaves.
