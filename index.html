ReTreever: Hierarchical Retrieval at Scale

Hierarchical retrieval that keeps the interpretability of tree-based search without giving up the speed of dense retrieval.

Motivation

Dense retrieval is fast, but opaque. Hierarchical retrieval is interpretable, but slow.

ReTreever learns a binary tree directly over embeddings. The hierarchy is optimized for retrieval, not clustering, so it can produce strong nearest-neighbor representations while exposing the semantic paths used during retrieval.

01

Embeddings only

No LLM calls are required for tree construction or navigation.

02

Coarse-to-fine

Any tree level can be used as a representation.

03

Inspectable paths

Queries and documents can be traced through the learned tree.

04

Scalable search

Single-index and multi-index retrieval both come from one learned hierarchy.

Benchmark

What ReTreever does

A split function at each node outputs a soft left/right routing probability. Probabilities multiply along a path to produce a distribution over leaves. A bidirectional contrastive objective pulls positive query-reference pairs toward similar leaf assignments.

1

Encode

Embed query and context with a frozen encoder.

2

Route

Pass embeddings through differentiable split nodes.

3

Match

Compare leaf-assignment vectors for retrieval.

4

Inspect

Read the path and semantic groupings.
