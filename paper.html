Hierarchical Retrieval at Scale

ReTreever is a retrieval-optimized hierarchical retriever that learns an inspectable binary tree over frozen encoder representations. It keeps the useful structure of hierarchical retrieval while avoiding LLM calls during tree construction and navigation.

Copy citationProject website

Problem

Flat dense retrieval is fast, but hard to inspect.

Standard dense retrieval stores items as high-dimensional vectors. It is effective, but it has large memory/compute footprints and gives little human-readable structure for understanding the retrieval process.

Solution

A retrieval-optimized tree over embeddings.

ReTreever learns a binary tree end-to-end. Query and context embeddings are routed through split nodes into leaf-assignment vectors; intermediate levels become coarser representations.

Figure 1. A frozen bi-encoder creates query and context embeddings. A differentiable tree organizer routes both through split nodes, producing leaf assignments for matching and coarser intermediate representations for cheaper retrieval.

Interpretability

Semantic groupings emerge naturally.

The learned hierarchy can be inspected. In Natural Questions, a subtree organizes content into coherent media-related topics such as Publishing, TV, Manga Comics, TV series, and Television seasons.

Scale

Stress-tested at 124.5M documents.

On Reddit Title-Body, the learned hierarchy provides a memory-friendly multi-index alternative to global ANN search: ScaNN exceeded 1 TB RAM, while ReTreever built a 1024-dim multi-index in 14 minutes.
